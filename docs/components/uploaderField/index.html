Uploader 上传组件

配置​

该字段配置在schema中：

schema
{
    "type": "object",
    "title": "上传组件",
    "ui": {
        "type": "uploader",
        "listType": "picture-card",
        "withCredentials": false,
        "canDrag": false,
        "accept": "https://www.jd.com",
        "action": "https://www.jd.com",
        "maxFileNumber": 5,
        "defaultFileList": true,
        "method": "get",
        "disabled": false
    },
    "fieldKey": "uploader_sDVWUt"
}
提示

组件具体配置依赖于所使用的UI库，DripForm仅作示例，具体可参考UI库对应文档。

API​

参数说明类型是否必填默认值
filedKey表单数据存放字段（必须唯一）string必填-
options可选数据源options[ ]必填-
action上传的地址string必填-
accept接受上传的文件类型string必填-
canDrag能否拖动boolean非必填false
defaultFileList默认已经上传的文件列表object[ ]非必填-
disabled值为 true 时，上传组件为禁用状态boolean非必填false
listType上传列表的内建样式，支持三种基本样式 text, picture 和 picture-cardstring非必填text
maxFileNumber限制上传数量。当为 1 时，始终用最新上传的文件代替当前文件number非必填1
method上传请求的 http methodstring必填Post
onChange上传文件改变时的状态function非必填-
withCredentials上传请求时是否携带 cookieboolean非必填false
